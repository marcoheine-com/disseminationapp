Automated Dissemination Application

As a part of the masterthesis "development of a communication and information concept for the dissemination of scientific results and publications" this automated application was developed to improve the dissemination of scientific results and publications from the IRIXYS Center.

It has been shown, that the dissemination of scientific publications on the web can be very successful and should not be neglected.

This website demonstrates the usage of this automated application as a prototypical implementation. It consists of three essential parts.

  • Social Networks for Scientists
  • The APIs of these Social Networks
  • Automated Notifications

The usage of social networks for scientists

The development of the Web 2.0 [1] showed, that any consumer of media can become a producer of media [2]. Besides new tools and technologies were developed, which supported this process, like Social Networks, Web- and Microblogs, Podcasts and Wikis. [2]

These processes and tools were transfered to the area of science communication. The web has always been an invention to improve communication between scientists [3], therefore it seems reasonable that Web 2.0 tools could be used successfully to improve science communication.

In this case one of the tools primarly used are ResearchGate, Mendeley and Academia.edu. These are Social Networks for scientists where scientists can create profiles for themselves, to share information about their education and their research interests.

Additionally they can connect with other scientists by following their profiles and build a network based on the same research interests. But the main feature is, that scientists can upload and disseminate their own publications on these platforms and directly adress the peers, which are interested in their work.

This describes the first step and the basis of this automated dissemination application. Every scientist is encouraged to use Social Networks for scientists regurlarly and upload and share his or her publications on this platforms.

The usage of the APIs/Data of these social networks

If every scientist frequently updates his social network profiles with his latest publications, the APIs of these Social Networks can be used to automatically grab the data und deploy it on the website of the science institute.

So whenever a scientist publishes a new paper or research results on platforms like ResearchGate, Mendeley or Academia.edu, it not only gets in the eyes of other scientits on these platforms, it also gets directly disseminated on the main website of the science institution.

This enables the possibility of providing a self-updating and always up-to-date collection of all research results and publications of the members of a science institute.

Additionally, the publications on the website can be searched and several statistics about the publications are also provided.

Automated Notifications with Twitter and the Web Notifications API

Social Networks for scientists are not the only way to practice a modern science communication. As described before, the website of a science institute itself can also be used to improve the dissemination of scientific publications.

But there is another tool, which became known with the rise of web 2.0: the microblogging service Twitter. Twitter provides a fast and easy way to share links to more extensive content, like research results or publications [4] [5], therefore it is of great use concerning science communication and the dissemination of scientific publications.

Twitter will be the last step of this automated application. When the data is loaded via the APIs of the Social Networks, firstly every publication gets stored as an object in an array locally and pushed to the localstorage of the users browser and secondly every publications is sent to the server and stored there.

On every API call there is a check, if the loaded publications are new or already existing in the localstorage and on the server. If they are completely new to the localstorage a web notification is sent out. If they are completely new to the storage on the server a new tweet is created and sent, using the Twitter API, to share the new publication on Twitter automatically.

What type of API the Social Networks offer and what possibilities they enable to support this automated application for the dissemination of scientific publications, is shown on the specific sub pages.

Unfortunately Academia.edu does not offer a an API or another way to fetch the data of a user account, so it can not be used with this automated application.
